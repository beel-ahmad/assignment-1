5: To check the git status

$ git status
RESULT: On branch master
No commits yet
Untracked files:
(use "git add ..." to include in what will be committed) git commands and notes.txt nothing added to commit but untracked files present (use "git add" to track)

6: To transfer the file from untracked to Staged

1.git add "git commands and notes.txt"

7: To commit the staged files

git commit
RESULT:
Vim editor will be opened
1: press i to INSERT
2: Write the commit notes ("Initial Commit")
3: Press 'ESC' and now type ':wq' and press Enter
4: Now run 'git status' command



8: To add more then one files from untracked to staged then press

git add -A
1. Now if you check the status you will see that files are now in staged area and now you can commit the files to shift them in unmodified phase.
2 Note: if you edit a already have a file in unmodified phase and edit it then it is now in modified phase and not you will have to shift to staged and then commit phase to take it back to the unmodified phase.

9: git commit -m "Enter your message here about the commit"

It will commit the changes without opening the vim editor

10: To take your modified file to the last commit stage (like :Ctrl+Z)

1.git checkout Filename
2.git checkout -f (To checkout all files at once)

11: To view All the commits

git log //show all the commits (press q to exit from git log)
git log -p -5 note: // show last 5(or any number you write) commits and also show the changes that were made It will commit the changes without opening the vim editor

12: Compare the working stage from the staged area

git diff
git diff --staged // compared staged stage with last commit
